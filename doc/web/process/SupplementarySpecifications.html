Introduction

Purpose

This document describes the system requirements that are not readily described in the use cases of the use-case model.

Scope

This document applies to the UFO@home project. It affects the overall requirements and design of the RR0 Information System.

Definitions, acronyms and abbreviations

  • API — Application Programming Interface.
  • Gb — Giga byte.
  • HTML — HyperText Markup Language.
  • J2EE — Java 2 Enterprise Edition.
  • J2SE — Java 2 Standard Edition.
  • JDO — Java Data Objects.
  • RAM — Random Access Memory.
  • UML — Unified Modeling Language.

References

  1. Vision

Functionality

  1. The system shall allow users to remain anonymous
  2. The system shall support (allow to provide, process) on unprecise data (unprecise or releative distance, time, etc.) because their commonness in ufological data

Usability

  1. Users willing to report a sighting shall not require any special training other than basic web browsing ability.
  2. Internationalization : Users willing to report a sighting shall be able to read and enter information in their own language.
  3. Users willing to report a sighting shall not spent a time longer than they would by reporting by hand.
  4. Users willing to report a sighting shall not require any other equipment or configuration than standard and basic web browsing equipment and configuration

Reliability

  1. The system shall be available 7 days per week, 24 hours per day.

Performance

  1. Response time for a transaction shall be less than 5 seconds.
  2. The system shall accommodate up to 500 concurrent users.

Supportability

  1. The system shall use standard Java APIs only.

Design constraints

  1. The system shall be described using the UML formalism.
  2. The system shall be implemented using the Java programming language.
  3. The reference implementation of the system shall implement its persistence using JDO.
  4. The system shall be J2EE compatible.

Online user documentation and help system requirements

  1. On-line user help shall be available for all tasks.

Purchased components

  1. A JDO implementation shall be acquired for the reference implementation. Depending on available funds/pricing, an free implementation shall be used or not.

Interfaces

This section defines the interfaces that must be supported by the application. It should contain adequate specificity, protocols, ports and logical addresses, etc., so that the software can be developed and verified against the interface requirements.

User interfaces

  1. The user interface shall be Web-based (HTML).

Hardware interfaces

  1. A server with at least 1 Gb of RAM shall be provided.

Software interfaces

  1. The system shall provide import/export interface for exchanging data with other UFO@home-compatible servers through any connected TCP-IP network, including the Internet.

Communications interfaces

  1. An Internet network connection with sufficient bandwidth shall be provided.
