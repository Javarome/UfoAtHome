Introduction

Purpose

This document is used to define terminology specific to the UFO problem domain, explaining terms which may be unfamiliar to the reader of the use-case descriptions or other project documents. This document can be used as an informal data dictionary, capturing data definitions so that use-case descriptions and other project documents can focus on what the system must do with the information.

Scope

This document applies to the UFO@home project. It affects the overall requirements and design of UFO@home.

References

  1. RR0 (french, translatable) — An amateur UFO website that contains various definitions and documentation about the UFO domain
  2. Javarome.net (french, translatable) — An amateur website that contains various definitions and documentation about Java software development.

Overview

The terms of the Definitions chapter form the essential substance of the document. They are defined by category, then in alphabetic order.

Definitions

Project

RR0

Rencontre Rapprochée 0. French acronym for Close Encounter of the kind 0 (CE0), which means that nothing was sighted at all. Historical name of a static ufological website that launched the UFO@home project.

UFO@home

The UFO@home system, code-named "RR0-IS".

RR0 IM

UFO@home Information Model. The set of concepts manipulated in ufology, as they will be used by UFO@home.

Stakeholder

A representative of people that can be affected by the existence of UFO@home (including UFO@home users).

User

People that will use the UFO@home product, once finished.

Business domain

AIAA

American Institute of Aeronautics and Astronautics.

In November 1970 the AIAA UFO subcommittee released in Aeronautics and Astronautics (Vol. 8. No. 11, P. 49-51) an article entitled UFO: An Appraisal of the Problem.

Case

A major ufological (sighting, abduction, etc.) event that required investigation

Category

A set of criteria that match a range of data.

Classification — A named set of mutually exclusive categories.

UFO

Unidentified Flying Object. Some unexplained aerial phenomena, sometimes with a high level of strangeness.

UFO phenomenon

The whole set of unexplained and maybe interrelated events involving UFOs.

Ufological

Related to ufology.

Ufology

Study of the UFO phenomenon.

Technical

Applet

(Litteraly "small application") A small software program that executes/displays into a Web page. For instance, UFO@home defines an applet that allow a witness to describe visually its sighting through the Internet.

API

Application Programming Interface. A set of software interfaces (sometimes classes) and associated operations (methods) that are visible to the caller of a software component. An API is written in/for a given programming language, like Java.

Examples of API are the JDO (Java Data Objects) standard Java API or the RR0 IM (UFO@home Information Model) Java API.

CRUD

Create, Retrieve, Update, Delete. Basic database operations.

HTML

HyperText Markup Language. The language to describe/write Web pages sent by a Web server to a Web client (browser). The HTML is maintained by the W3C (World Wide Web Consortium).

HTTP

HyperText Transfer Protocol. The network protocol between a Web browser and a Web server. Runs over the TCP/IP protocol.

Implementation

The action of writing a software program. Also used to qualify the result of such an action, that is, the written software itself.

Interface

An element that allows to define some interaction between two other elements. For instance, UFO@home defines some "user interfaces" that allow some people to interact with some software components such as the sighting report applet or a form to fill on the Web. Also used to describe interactions between purely software element purement, such as two software objects (for instance there is an interface to describe the relationship between witness and testimonial objects). UFO@home defines a whole set of such software interfaces, and let people free to provide implementations of those interfaces.

J2SE

Java 2 Standard Edition. The standard software platform for executing Java programs.

JDO

Java Data Objects. A standard Java API to allow Java objects to persist transparently in any datastore (RDBMS, ODBMS, etc.).

UUDF

Uniform Ufological Data Format. An XML-based data format aimed to describe any ufological data in a standardized way compatible with Web services.

UML

Unified Modeling Language. A standard formalism that defines diagram types and diagrams symbols to represent requirements, analysis, design and deployment semantics in a standardized way.

XML

Extensible Markup Language. A language of user-defined hierarchical tags.

Web service

A specification for interoperability (data exchange) between application through the Internet.

W3C

Word Wide Web Consortium. The organization that defines standards for the Web.
