Introduction

Purpose

The purpose of this document is to collect, analyze and define high-level needs and features of UFO@home. It focuses on the capabilities needed by the stakeholders, the target users, and why these needs exist. The details of how the UFO@home fulfils these needs are detailed in the use-case and supplementary specifications.

Scope

This Vision Document applies to the UFO@home project, developed by volunteers of the Open Source community. These volunters will develop this information system to allow witnesses, researchers and investigators on the UFO phenomenon to interface with an existing, shared and centralized UFO database.

Positioning

Business opportunity

After almost 60 years of its modern era, ufology has produced several ufological databases, each of these managing their own data set, either public or private, according to ad hoc business (data structure, classification) and technological (storage, import/export formats, running platform) rules. The same schema can be found among static Web amateur websites devoted to the UFO subject.

UFO@home aims to be a tool for a long-running study of the UFO phenomenon, available to the whole ufological community. As stated in [1], there is little doubt that the short-time, one-shot approach of an ad hoc team is neither promising nor economical. This is especially true if the study team decides (...) to concentrate on current rather than past observations. As the UFO statistics show, this results in the devotion of precious time to investigating the noise, rather than the signal. (...). There is also little hope to expect a solution of the extremely complex problem by the efforts of a single individual. It notably aims to allow data access according to standard principles and formats. As stated in [6], In his end chapter, Hynek suggests making a new start on the whole UFO problem. He recommends a statistical approach to define the parameters of the phenomenon. This implies computerized data processing and an international agreement on formats and coding.

UFO@home aims to build a platform allowing a more collaborative work, along 3 axes :

  • Sighting : data input/browsing
  • Process : Data processing (statistics, filtering, etc.)
  • Investigation management/follow-up.

Data processing

The statistical approach frees itself of specificities (or bias) of each invidual case (as amazing this case could be) to attempt an extraction of patterns, structures or underlying functional relationships inside or between some data groups. Beyond statistical analysis :

  • descriptive statistics — moments, probability density, outliers
  • multivariables linear/non-linear regression
  • autoregressive modeling
  • Markov modeling
  • analysis for main components
  • factorial analysis
  • partitioning/clustering — hierarchical trees, dendrograms
  • statistical inference and Bayesian classification
  • statistical hypothesis test
  • spectral analysis

exists a number of heuristics and algorithms out of domains related to statistics (information processing, artificial intelligence) used for instance by genomics or by advertisement vendors (which leverage huge files to isolate preferences, consumption behaviors and so better target their advertisements) :

  • data-mining
  • adaptative clustering techniques
    • évolutionnist (genetics) and artificial life algorithms
    • neuronal networks
    • semantic networks

from which we will be able to discover any applicability to our case through a bibliographic study (as a first step).

As those techniques use to work a priori without any physical, physiological or psychosocial model, one important modeling effort will certainly be undertaken in these areas to provide more "visibility" to underlying structures which could be meaningful and which we target to show up. It is the more interesting, wide and trans-disciplinar work.

A last reflexion will have to consider the most interesting (informative, controllable complexity : reliable) representations and visualisations for UFO@home.

Problem statement

The problem of disparity in UFO knowledge management.
affects investigators, scientists, witnesses
The impact of which is
  • difficulty in sharing information : there is not existing ufological data exchange format between researchers. Since the amount of relevant data is enormous, each research effort has to build from scratch its own database to process it as its will.
  • difficulty in consolidating data : disparity in ufological data contents and formats (missing or additional information, different classifications, etc.) lead to differents research results, depending on the data sources used.
  • credibility issues : multiplying databases (with differents quality, organization) gives a unorganized and immature image of the ufological community which discourage serious and scientific researchers. As Vallee states in [2], The best UFO data has never been published. I think a great deal of the misunderstanding about UFOs among scientists is that the scientists have never had access to the best data.
  • difficulty in accessing to the information because of the spreading of varying databases and the required technological means (installing a local database, online database). Private or commercial databases also limit the ability to study the phenomenon.
A successful solution would standardize/unify exchange, processing and formats of ufological data.

Stakeholder and user descriptions

This section describes the users of UFO@home. There are several types of users of the UFO@home; the Witnesses, the Researchers, and the Administrators.

Market demographics

To effectively provide a product and services that meet the stakeholders’ and users' real needs, it is necessary to identify and involve all of the stakeholders as part of the Requirements Modeling process. We must also identify the users of the system and ensure that the stakeholder community adequately represents them. This section provides a profile of the stakeholders and users involved in the project, and the key problems that they perceive to be addressed by the proposed solution. It does not describe their specific requests or requirements as these are captured in a separate stakeholder requests artifact. Instead, it provides the background and justification for why the requirements are needed.

The ufological community is large and very heterogeneous. It includes journalists, writers, military people, scientists, editors and producers, and UFO amateurs. Some of these people that actively lead research about UFOs sometimes call themselves "ufologists".

Journalists interested in ufology represent a small portion of the whole journalism professionals community. In this small population can be found active and passive people. While the former regularly participate in the ufology chronicle and so knows the phenomenon very well, the latter consider the UFO subject either as triggering an interesting sociological phenomenon or as a sensational subject that can bring up some readers when other exciting news can be found. However those people usually follow a method, and have some rigour and objectivity that can help in studying the phenomenon.

Writers representing a tiny but valuable portion of the UFO community. are more often dependent on the ufological chronicle to live. They used to be "professional" researchers in the subject, and so have a very comprehensive knowledge of the UFO phenomenon and can be among the better "specialists" of some UFO cases. Conversely, producers, editors and some journalists, can be accused of promoting sensationalism, using topics like UFOs.

Scientists represent a tiny population in the UFO community. They can be divided between a minority of people with their minds opened to the subject, and people that don't intend to pursue any research in the area, mostly because of lack of funding, and/or the fear of ridicule. They can however be very valuable to UFO research because of their methodology and so recognized credibility.

Military people represent a tiny population of the UFO community. People from this population often shows interest in ufology publicly only at the end of their career, providing an inside, but often outdated view of the phenomenon from a military point of view. Some military people are interested in the UFO phenomenon because of its psychological warfare aspects, but also to "tell the truth" about past unidentified aerial phenomena.

Producers and editors are usually interested in the commercial aspect of the UFO hype. Because of this they can cause some damage to the field's credibility but have also the ability and some talent to spread information to a wide audience and occasionally fund some research underlying a commercial product. Examples are the Sci-Fi Channel, Ray Santili, Jaime Shandera, etc.

UFO amateurs represent the large majority of the UFO community. They are heterogeneous in field experience, rigor and education, but clearly the most active ones. They are quite unorganized.

All of these The users can be educated, computer literate, and often own personal computers in their homes. The ability to report a sighting via personal computers and to review existing cases would greatly streamline the ufological information system.

To meet the potential amount of demands, UFO@home servers will have to be hosted by professionnal service providers. Funds may be needed to do so.

Precedence and priority

This section provides some direction on the relative importance of the proposed system features. The features defined in this Vision Document should be included in the first 2 releases of the system. All features critical to sighting reporting are planned for the first release.

As development progresses on this system, the feature attributes will be used to weight the relative importance of the features and to plan the release content. The benefit, effort, and risk attributes are used to determine priority of a feature and target release.

It is anticipated that the UFO@home Information System will be released for general use through 2-4 main releases.

Release 1 must contain as a minimum the basic functionality as listed below :

  • Logon
  • Report a sighting
  • Adding/updating a case
  • Searching for a case

Release 2 should include :

  • Data processing (statistics, classification functions, etc.)

Release 3 should include :

  • UUDF import/export
  • Interactions between multiple UFO@home-compatibles servers.

The functionality for Release 4 has not yet been determined. It is anticipated that this release will contain enhancements to the existing functionality.

Other product requirements

Applicable standards

  1. The UFO@home user interface shall be W3C HTML 4.0 compliant.
  2. The UFO@home network protocols shall run over HTTP (to allow interoperability through Internet)
  3. The UFO@home application shall be compatible with the Java 2 Enterprise Edition (J2EE) platform.

System requirements

  • The system shall interface with other UFO@home servers.
  • The client component of the system shall operate on any personal computer with a Pentium Microprocessor or better.
  • The client component of the system shall not require more than 256 MB RAM and 20 MB Disk Space.
  • The client component of the system shall run on Microsoft Windows 95, Windows 98, Windows NT, Windows 2000 and Windows XP, Linux (non restrictive list).

Performance requirements

  • The system shall support up to 500 simultaneous users against the central database at any given time.
  • The system shall provide access to the legacy UFO@home database (except functions) with no more than a 5 second latency.
  • The system shall complete 80% of all transactions within 2 minutes.

Environmental requirements

None specified.
