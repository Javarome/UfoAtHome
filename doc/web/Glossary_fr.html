Introduction

Objectif

Ce document est utilisé pour définir la terminologie spécifique au domaine ufologique, expliquant des termes qui pourraient être non familiers au lecteur des descriptions de cas d'utilisation ou d'autres documents du projet. Ce document peut être utilisé comme un dictionnaire de données informel, capturant les définitions de données de manière à ce que les autres documents du projet puissent se focaliser sur ce que le système doit faire de l'information.

Périmètre

Ce document s'applique au projet UFO@home. Il affecte l'ensemble des besoins et de la conception de UFO@home.

Références

  1. RR0 — Site web amateur contenant de nombreuses définitions relatives du domaine ufologique.
  2. Javarome.net — Site web amateur contenant de nombreuses définitions et documentations sur le développement logiciel et Java.

Vue d'ensemble

Les termes du chapitre de définitions suivant forment la substance essentielle de ce document. Ils sont définis par catégorie, puis par ordre alphabetique.

Définitions

Projet

RR0

Rencontre Rapprochée 0. Nom historique d'un site web ufologique qui a initié le projet UFO@home.

RR0 IM

RR0 Information Model : Modèle d'Information de UFO@home. Ensemble de concepts manipulés en ufologie, tels qu'il vont être utilisés par UFO@home.

Représentant

Ou Stakeholder. Personne affectée par l'existance de UFO@home. Inclut les utilisateurs de UFO@home.

Utilisateur

Personne qui utilisera le produit UFO@home, une fois achevé.

Domaine métier

AIAA

American Institute of Aeronautics and Astronautics : Institut Américain d'Aéronautique et d'Astronautique.

En Novembre 1970 le sous-comité OVNI de l'AIAA publie dans Aeronautics and Astronautics (Vol. 8. No. 11, P. 49-51) un article intitulé UFO: An Appraisal of the Problem.

Cas

Un événénement ufologique majeur (observation, enlèvement, etc.) qui nécessite ou a nécessité une enquête.

Catégorie

Un ensemble de critères correspondant à une selection de données.

Classification

Un ensemble nommé de catégories mutuellement exclusives.

OVNI

Object Volant Non Identifié. Un phénomène aérien inexpliqué, parfois avec un haut degré d'étrangeté.

UFO

Unidentified Flying Object. Equivalent anglophone d'OVNI.

Phénomène OVNI

L'ensemble des événements inexpliqués et peut-être liés impliquant des OVNIs.

Ufologique

Relatif à l'ufologie.

Ufologie

Etude du phénomène OVNI.

Technique

Applet

(Littéralement "petite application") Petit programme s'exécutant/s'affichant dans une page Web. UFO@home définit par exemple une applet permettant à un témoin de décrire visuellement son observation via Internet.

API

Application Programming Interface. Interface de Programmation d'Application. Ensemble d'interfaces (parfois de classes) et d'opérations associées (méthodes) visibles de l'appelant d'un composant logiciel. Une API est écrite dans/pour un langage de programmation donné, comme Java.

Des exemples d'API sont le standard Java JDO (Java Data Objects) ou l'API Java RR0 IM (UFO@home Information Model).

CRUD

Create, Retrieve, Update, Delete. Création, Récupération, Mise à jour, Suppression. Opérations basiques sur une base de données.

HTML

HyperText Markup Language. Langage de balises pour l'hypertexte. Le langage pour décrire/écrire des pages Web pages envoyées par un serveur Web vers un client Web (navigateur). L'HTML est maintenu par le W3C (World Wide Web Consortium).

HTTP

HyperText Transfer Protocol. Protocol de transfert de données hypertextes. Le protocole réseau entre un navigateur Web et un serveur Web. Situé au-dessus du protocole TCP/IP.

Implémentation

(Anglicisme, parfois traduit en français par "implantation"). Action d'écrire un programme. Egalement utilisé pour décrire le résultat de cette action, c'est-à-dire le programme lui-même.

Interface

Elément permettant de définir une interaction entre deux autres élements. Par exemple UFO@home définit des "interfaces utilisateur", permettant à une personne d'interagir avec des éléments logiciel comme l'applet de témoignage d'ovni ou un formulaire à remplir sur le Web. Egalement utilisé pour décrire des interactions entre des élements purement logiciels, tels que deux objets informatique (il existe une interface pour décrire la relation entre un objet témoin et un objet témoignage par exemple). UFO@home définit tout un ensemble d'interfaces logicielles, et laisse la liberté à tous de fournir des implémentations de ces interfaces.

J2SE

Java 2 Standard Edition. Edition Standard de Java 2. La plate-forme logicielle standard pour exécuter des programmes Java.

JDO

Java Data Objects. Objets de données Java. Une API Java standard permettant à des objets Java de persister de manière transparente dans tout support de stockage (SGBDR, SGBDO, etc.).

UUDF

Uniform Ufological Data Format. Format de données ufologiques uniforme. Un format de données basé sur XML visant à décrire toute donnée ufologique d'une manière standard et compatibles avec les serveurs Web.

UML

Unified Modeling Language. Langage unifié de modélisation. Un formalisme standard définissant des types de diagrammes et des symboles pour représenter les sémantiques de besoin, analyse, conception et déploiement de manière standardisée.

XML

Extensible Markup Language. Langage extensible de balises. Un langage de balises hiérarchiques personnalisables.

Service Web

Une spécification pour l'intéropérabilité (l'échange de données) entre application via Internet.

W3C

Word Wide Web Consortium. Consortium du Web mondial. L'organisation qui définit les standard du WebWeb.
