The Problem
  • Ufological data are :
    • Not reusable
      • No standard way to
        • describe data (sighting, investigation, etc.)
        • To link data
    • Not shareable
      • Results can really be reviewed
      • Multiple databases and formats
      • Differences in data amounts and contents
      • Every new project is a new incompatible database
        • This can’t be avoided

Vision benefits
  • View all the data
    • non only yours
  • Process all the data
    • Search/filtering, classifying, statistics
    • Build and share your own processes
  • Processes are long lived
    • They depend on the metamodel
      • Not the concrete databases anymore
  • Results can be shared
  • Old databases can migrate, not just die
8
The Need : Traps About Data
  • Selecting data
    • Researchers’ job
      • Understanding a unknown phenomenon requires all the data
  • Interpreting data
    • Researchers’ job
      • the tool only helps (customizable processing)
  • Hiding data
    • User’s preferences
      • But data origin (sources, authors) is critical for metadata
  • Centralizing data
    • Anti-trust accusations
    • People don’t build using a single hammer
9
The Need : Traps About People
  • Selecting people
    • Witnesses are not to be selected
    • We need both amateur and scientists
  • Build a software that impose too much constraints
    • Ease of use is critical for a de facto standard
      • For witnesses
      • For contributors
    • Don’t impose data
    • Don’t impose processes

Deployment
  • The UFO@home platform can run on any host
    • Any hardware (PC, Mac)
    • Any OS (Windows, Linux, MacOS)
  • Any deployment
    • Does not mandate data centralization
      • Actually most is from legacy database that doesn’t move
    • Allow replication
      • For performance, data safety
    • Client-Server, Distributed, Standalone, Peer-to-peer
