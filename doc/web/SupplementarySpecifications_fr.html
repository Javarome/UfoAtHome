Introduction

Objectif

Ce document capture les besoins pour le système qui ne sont pas aisément exprimables au travers des cas d'utilisation du modèle de cas d'utilisation.

Périmètre

Ce document s'applique au projet UFO@home. Il affecte l'ensemble des besoins et de la conception de UFO@home.

Definitions, acronymes et abbréviations

  • API — Application Programming Interface.
  • Go — Giga Octet.
  • J2EE — Java 2 Enterprise Edition.
  • J2SE — Java 2 Standard Edition.
  • JDO — Java Data Objects.
  • RAM — Random Access Memory. Mémoire vive.
  • UML — Unified Modeling Language.

Références

  1. Vision

Fonctionalité

  • Le système devra permettre aux utilisateurs de conserver leur anonymat.
  • Le système devra supporter (permettre de fournir, traiter) des données imprécises (distance ou temps imprécis ou relatifs, etc.) en raison de leur fréquence dans les données ufologiques.

Utilisabilité

  1. Les utilisateurs souhaitant signaler une observation ne devront pas nécessiter d'autre formation spéciale que de savoir surfer sur le Web.
  2. Internationalisation : les utilisateurs souhaitant signaler une observation devront pour lire et saisir de l'information dans leur propre langue.
  3. Les utilisateurs souhaitant signaler une observation ne devront pas y passer plus de temps que s'ils le faisaient à la main.
  4. Les utilisateurs souhaitant signaler une observation ne devront pas nécessiter d'autre équipement ou configuration que l'équipement et la configuration standard pour surfer sur le Web.

Fiabilité

  1. Le système devra être disponible 7 jours par semaine, 24 heures par jour.

Performance

  1. Le temps de réponse pour une transaction devra être inférieur à 5 secondes.
  2. Le système devra supporter jusqu'à 500 utilisateurs simultanés.

Supportabilité

  1. Le système ne devra utiliser que les API Java standard.

Contraintes de conception

  1. Le système devra être décrit en utilisant le formalisme UML.
  2. Le système devra être implémenté avec le langage de programmation Java.
  3. L'implémentation de référence devra implémenter sa persistance en utilisant JDO.
  4. Le système devra être compatible J2EE.

Besoins pour la documentation et le système d'aide en ligne

  1. Une aide en ligne devra être accessible pour l'ensemble des tâches.

Composants achetés

  1. Une implémentation JDO devra être acquise pour l'implémentation de référence. En fonction des tarifs et des fonds disponibles, une implémentation gratuite pour être utilisée ou non.

Interfaces

Cette section décrit les interfaces qui devront être supportées par l'application. Elle doit contenir les spécificités, protocoles, ports et adresses logiques, etc. adéquats pour que le logiciel puisse être développé et que l'on puisse vérifier sa confirmité aux besoins d'interfaces.

Interfaces utilisateur

  1. L'interface utilisateur devra être une interface Web (HTML).

Interfaces matérielles

  1. Une serveur avec au moins 1 Go de RAM devra être fourni.

Interfaces logicielles

  1. Le système devra fournir une interface d'import/export pour échanger des donner avec d'autres serveurs compatibles UFO@home via une connection TCP-IP, y compris Internet.

Interfaces de communication

  1. Une connection Internet avec une bande passante suffisante devra être fournie.
