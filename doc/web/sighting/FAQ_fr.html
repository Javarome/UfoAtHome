Pourquoi une représentation 2D ?

L'applet de témoignage visuel vise à représenter ce que le témoin a vu. C'est littéralement une reconstitution visuelle de l'image qu'a reçue le témoin, qui est en 2 dimensions par nature. Le ou les objets observés peuvent être tridimisionels, mais ce type d'interprétation reste le travail des chercheurs, et ne fait pas partie des objectifs de l'outils.

De plsu, notez que l'applet vise également à représenter en 3 dimensions ce qu'elle sait être en 3 dimensions avec certitude, c'est-à-dire la sphère céleste composée des étoiles, planètes et de la Lune, ainsi que l'environnement direct connu du témoin (routes, bâtiments, etc.).

Pourquoi une applet Java ?

Java est un langage de programmation mais aussi une plate-forme d'exécution logicielle disponible sur de nombreuses machines ((PC, assistants personnels, gros systèmes, etc.), et systèmes d'exploitation (Windows, Unix, etc.). Une applet un est programme Java côté client capable de s'exécuter dans des navigateurs supportant Java, tels que Microsoft Internet Explorer, Netscape Navigator, Mozilla, Opera, etc.

Limiter le code de l'applet de témoignage visuel à Java 1.1 lui permet de s'exécuter dans les navigateurs les plus populaires du marché (dont Microsoft Internet Explorer) sans nécessiter de téléchager d'autre programmes ou plugins. C'est là un avantage critique répondant au besoin du projet de toucher le public le plus large possible. Se limiter à Java 1.1 (côté client) permet également au code d'être compatible avec des applications Java s'exécutant sur des appareils portables tels que les assistants personnels ou PocketPCs (i.e. J2ME CDC Personal Profile).
