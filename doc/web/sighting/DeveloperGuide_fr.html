Introduction

Objectif

Ce document vise à aider le développeur à implémenter, étendre et/ou modifier le système UFO@home.

Périmètre

Ce document s'adresse aux informaticiens (développeurs, ingénieurs, concepteurs) participant à l'élaboration technique d'un système UFO@home. Il concerne l'ensemble du développement du système UFO@home.

Définitions, acronymes et abbreviations

  • AWT — Abstract Windowing Toolkit. Une API Java pour construire une IHM qui puisse s'exécuter sur des applets Java sur le Web ou sur une CDC J2ME.
  • CDC — Connected Device Configuration. Une configuration J2ME supportant AWT.
  • GUI — Graphical User Interface.
  • IHM — Interface Homme Machine.
  • J2ME — Java 2 Micro Edition. Une spécification de plate-forme Java pour les appareils embarqués ou mobiles.
  • HTML — HyperText Markup Language.
  • HTTP — HyperText Transfer Protocol.
  • RR0 IM — UFO@home Information Model

Voir également le Glossaire du projet.

Références

  1. Javarome.net — Un site Web amateur contenant diverses définitions et documentation sur le développement Java.

Vue d'ensemble

Ce document décrit les divers point d'extension d'un système UFO@home, et la manière de les exploiter. Il est organisé selons les différents modules du système.

Modules clients

Contraintes de platforme

Les modules client sont restreints aux plates-formes suivantes :

  • HTML 4.01
  • applets Java 1.1.8 (i.e., pas au-delà), pour garantir la compatibilité avec Microsoft Internet Explorer (i.e. sans nécessiter d'autres composants à télécharger) et le Personal Profile (PP, anciennement appelé PersonalJava) de la configuration CDC de J2ME (ce qui permet de s'exécuter sur des appareils portables et PDAs such tels que des Palm/PalmPilots, PocketPCs, etc.).

Applet de témoignage visuel

Formes d'ovni

Chaque forme d'ovni est un JavaBean. Elle dispose de propriétés personnalisables, et se conforme à une interface particulière.

Pour créer une nouvelle forme d'ovni, créez simplement une nouvelle classe implémentant l'interface UFOShape.
