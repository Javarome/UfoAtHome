Technical architecture

Home  |  Contact

UFO@home is built on a 3-tier architecture, separating cleanly presentation, business and integration/data access layers. This will allow to support multiple client types, such as Web browsers, heavyweight GUI clients, and even other informations systems through Web services.

  1. un client Web (navigateur)
  2. un serveur applicatif, herbergeant une application divisée en couches de :
    • présentation : contrôle de la logique de navigation Web, construction dynamique des pages
    • services : actions sur des objets métiers
    • métier : objets du domaine, persistants ou non
    • integration : glue entre données en mémoire et données persistant dans le référentiel UFO@home
  3. le référentiel UFO@home (base de données ou autre)
